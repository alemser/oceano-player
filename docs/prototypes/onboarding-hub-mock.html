First-run checklist

Row states sync with the Hub → Simulate install state selector (simulates GET /api/setup-status). "Skip" is implicit (no state written); "Dismiss" stores a localStorage flag per row for explicit opt-outs (e.g. stylus for CD-only users).

  1. Run sudo oceano-setup on the Pi — DAC, capture card, AirPlay / BT basics
  2. Choose capture device — REC OUT → USB interface
  3. Add ACRCloud credentials — recognition provider
  4. Optional
  5. Set up amplifier — identity + inputs + Broadlink IR (or inputs-only, no IR)
  6. Run calibration for physical media inputs only (OFF/ON noise floor)
  7. Track stylus hours — shown when vinyl_topology_present is true; dismiss permanently if CD-only

Streaming (AirPlay name, BT) is a compact optional row after step 3.

Amplifier wizard

Do you want IR control of your amplifier?

If you choose No, you can still name inputs and attach turntable / CD / streamer so calibration scope, recognition context, and Now Playing labels stay correct — without Broadlink or IR codes.

No IR path: define maker / model or pick a built-in profile so input IDs and labels match your amp. Broadlink and IR learning are skipped. Next: connected devices and roles.

Profiles pre-fill inputs, warm-up, standby, and USB-reset timings. Inputs are editable in the next step.

Broadlink pairing — required before IR commands

The RM4 Mini must be reachable and paired (host + token / device ID persisted) before any IR learning can begin. The Next button below is disabled until pairing succeeds. You can use the existing pair.html wizard — a separate pairing flow is fine; this step gates the order.

Draft persistence: if you close the browser here and return, steps 1–2 are restored from localStorage so you do not re-enter identity and IR intent. IR-learned codes are only written on final Save.

Learn IR commands

Each row: Learn / ✓. Blocked until Broadlink pairing (step 3) is satisfied. Skippable rows labelled; power commands are not skippable.

Connected devices & roles

For each device: name, amplifier input(s), and role — Physical media / Streaming / Other. For physical media rows, set physical_format: Vinyl · CD · Tape · Mixed · Unspecified. When input is Phono and format is still unspecified, a single follow-up ("Turntable / vinyl?") sets vinyl without clutter. This drives calibration scope, stylus prompts, and Now Playing labels.

IR disabled. Broadlink pairing and IR learning were skipped. Map your connected devices (name, amplifier input(s), role, physical_format) so calibration scope, stylus prompts, and labels remain correct — no Broadlink required.

Implementation: same connected_devices config shape as the IR path; identical API.

Optional — Stylus tracking

Shown when physical_format === vinyl (confirmed turntable on Phono or explicit). User picks cartridge profile + rated hours in the existing amplifier.html stylus block or Listening Metrics. Hours accrue on Vinyl + Physical playback sessions. This CTA is available even when amplifier.enabled === false — vinyl listeners reach it without enabling IR.

Calibration scope (Phase 4)

Current implementation supports connected_devices[].is_turntable for explicit vinyl mapping (with phono/vinyl name fallback). Phase 4 extends this with role + physical_format, but the turntable path already solves AUX1-style setups.
Current UI location: auto-tuning from playback results and RMS percentile learning are configured in recognition.html, with advanced knobs collapsed. advanced.html stays focused on sockets/paths/library maintenance.
  • Calibratable inputs = union of input IDs on physical_media devices
  • Vinyl gap sub-step only when physical_format === vinyl
  • Streaming / Other inputs hidden from calibration list by default (manual override available)
  • Migration: missing role → physical_media (safe for existing configs); new devices via wizard must select role explicitly
  • CD calibration note: Off/On RMS differential is often too small on CD transports — copy should steer toward conservative vu_silence_threshold and the RMS histogram path instead of insisting on a wizard profile
  • RMS snapshot in UI is user-facing: shows CD and Vinyl; internal Physical learning remains background-only

Now playing — amplifier line (Phase 6)

Kiosk + mobile parity: the same read-only line renders on both HDMI kiosk (non-touch) and phone (coarse pointer). Touch-only affordance (input picker) activates only on coarse pointer / tap.

Magnat MR 780 · Rega Planar

When Phono is mapped to a single connected device, the chip shows device name only (no redundant "Phono"). Multi-input devices use Device · logical input. Same labelling rules as the remote input dropdown.

Switch input

Touch only (coarse pointer / phone). Kiosk always shows read-only line; this sheet never appears under mouse at 2–3 m.

Offline state: when Broadlink is unreachable, the input picker is hidden — the pill remains read-only, no error chrome. Graceful degradation.

Roadmap: optional read-only stylus chip (e.g. 12 h / 500 h rated) when vinyl topology + profile are configured — see plan → Stylus section.

Returning user — new amplifier / rewiring

Primary CTA: Change amplifier / re-run setup (opens the same wizard). Entry point is always visible on the hub regardless of setup status.
  • Broadlink: often still valid when swapping amps → "Already paired — continue" in step 3.
  • IR codes: almost always re-learn when the amp model changes.
  • Calibration profiles: keyed by amplifier input ID. If the new amp or profile changes IDs, surface a warning: "Phono was input 20, now 10 — re-run calibration for Phono." Old profiles for removed IDs can be cleared or kept.
  • Connected devices + roles: re-attach turntable / CD player to new input IDs in the wizard — keeps calibration filter and stylus accumulation accurate.
