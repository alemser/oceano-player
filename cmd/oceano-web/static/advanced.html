Auto-tuning from playback results

The state manager uses boundary_events follow-ups (matched vs same-track-restored) and applies small bounded adjustments to VU silence thresholds and duration pessimism. RMS time series are not logged per boundary row; RMS snapshots live in calibration_profiles from the wizard, while live RMS comes from the VU socket.

When on, adaptive tuning runs whenever paired follow-ups meet the minimum — you do not need the manual checkbox below. Does not auto-edit calibration JSON; only bounded runtime nudges.

RMS percentile learning

While Physical is active, the state manager records RMS into per-format histograms (stable silence vs stable music) in the library database and derives VU silence enter/exit from percentiles. When Autonomous apply is on, those thresholds override wizard profiles once sample counts are met. The manual wizard remains optional for transition (vinyl gap) metrics.
